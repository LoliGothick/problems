Trial 2

比較的小さい正の整数 N が素数であるかどうかを判定するためには j を
2 から N-1 まで増やして, N を j で割り算したあまりが 0 であるかを判定し,
一つでも 0 になったら素数でない, 最後まで 0 でなければ素数とする算法が使用できます.
しかし, この素朴な算法は無駄が多すぎて非効率的です.
割ってみる範囲を狭くして効率化をはかるには, どのように修正すればよいか.

以下のC++のコードは上記の素朴な算法を関数として記述したものである.
このコードを修正し, その方法と理由をコメント文で説明しなさい

bool is_prime(unsigned long long n){
  if(n<2) return false;
  for(unsigned long long j=2; j<n; ++j)
    if(n%j==0) return false;
  return true;
}
ヒント：
O(N)→O(√N)の改善が一つと
定数倍の改善策が２つあります
